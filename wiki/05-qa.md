## 已知问题

### BUG-001：筛选功能未实现
- **位置：** `ClipLibrary.tsx` 中的三个 Select 组件
- **现象：** 切换 SOURCE KOL / CATEGORY / SORT 下拉框，片段列表不变化
- **原因：** Select 的 `defaultValue` 固定，未绑定 state，未实现过滤逻辑
- **优先级：** 中

### BUG-002：Add Entity 按钮无响应
- **位置：** `KOLManager.tsx` 第 22 行
- **现象：** 点击"Add Entity"按钮无任何反应
- **原因：** `onClick={() => {/* Open add dialog */}}` 为空实现
- **优先级：** 高

### BUG-003：搜索结果为硬编码
- **位置：** `Search.tsx`
- **现象：** 无论输入什么查询词，结果始终相同（`clips.filter(c => c.relevance)` 只过滤有 relevance 字段的 clip）
- **原因：** 未接入真实语义搜索 API
- **优先级：** 高

### BUG-004：合成页时间轴初始化固定
- **位置：** `Combine.tsx` 第 9 行
- **现象：** 时间轴始终加载前 5 个 clip，无法从搜索结果或片段库添加
- **原因：** `useState<Clip[]>(clips.slice(0, 5))` 硬编码
- **优先级：** 中

### BUG-005：任务进度不实时更新
- **位置：** `TaskMonitor.tsx` + `store.ts`
- **现象：** running 任务的进度条固定在初始值（62%），不会自动推进
- **原因：** 无 WebSocket 或轮询机制，store 中无进度更新 action
- **优先级：** 高

## 改进建议

### 短期（功能补全）

1. **实现 Add KOL 弹窗** — 复用 editingKol 的 Dialog，新增"新建模式"
2. **接入筛选逻辑** — 将 Select 值绑定到 state，用 `useMemo` 计算过滤后的 clips
3. **实现 COPY REF** — 使用 `navigator.clipboard.writeText(clip.id.toString())`
4. **任务进度轮询** — 添加 `setInterval` 定期调用 `/api/jobs` 更新进度

### 中期（架构完善）

5. **接入 Gemini API** — 实现真实的语义搜索和片段摘要生成
6. **持久化存储** — 将 Zustand store 接入后端 API（Express 层已在依赖中）
7. **从片段库添加到合成** — 在 ClipLibrary 卡片上增加"Add to Timeline"按钮

### 长期（测试体系）

8. **单元测试** — 为 store actions（`triggerJob`、`updateKOL`）添加 Vitest 测试
9. **组件测试** — 使用 React Testing Library 测试关键交互（搜索、弹窗）
10. **E2E 测试** — 使用 Playwright 覆盖核心用户旅程（添加 KOL → 触发任务 → 查看片段）
